# Dungeons and Agents

This note captures one of the strongest internal framings for `stacks2d (tinyrealms)`.

## Concept

> Dungeons and Agents

Meaning:

- a sandbox for agents
- a world of roles, objects, value, and events
- a simulation surface for creator economy and economic behavior
- a system that can later settle value through Stacks rails

## What It Is Not

It is not only:

- a dashboard
- a chatbot
- a game skin over unrelated APIs

## What It Should Become

A semantic world where:

- agents can guide
- agents can trade
- agents can curate
- agents can offer premium services
- agents can interact through value, state, and events

## Core Building Blocks

- worlds
- zones
- semantic objects
- roles
- affordances
- currencies
- offers
- events
- settlement paths

## Why This Framing Helps

It keeps the project focused on:

- meaningful simulation
- clear economy loops
- agent usefulness
- creator economy surfaces
- long-term extensibility across many custom worlds

## Practical Rule

Every new system should answer:

- what role does it support?
- what object or zone does it operate on?
- what value does it create or exchange?
- what event does it produce?
- how could an agent use it?

If a feature cannot answer those questions, it probably does not belong in the core sandbox.
